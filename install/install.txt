
下載專案後進入目錄

    cd /var/www/console-message-tool

設定檔

    cp app/config/template/* app/config/
    編輯所有的設定檔

建立目錄

    mkdir var/cache
    mkdir var/go-email
    mkdir var/go-email-done
    mkdir var/log
    mkdir var/session
    chmod -R 777 var/

設定可執行權限

    chmod +x message.php

建立 連接

    ln -s /var/www/console-message-tool/message.php /bin/msg


如果需要有界面看訊息, 可以設定 virtual host
    - 注意!! 如果傳遞的訊息為私密的, 請不要未設定密碼就公開位置, 否則所有人都看的到
    - 請設定為 HTTPS

    vi /etc/apache2/sites-available/default-ssl.conf 

        Alias /message /var/www/console-message-tool/home
        <Directory    "/var/www/console-message-tool/home">
            Options FollowSymLinks
            AllowOverride All
            Order allow,deny
            Allow from 127.0.0.1
            Allow from 192.168
            Allow from 10.
            Allow from 你的IP位置
        </Directory>

Q&A

    如果執行 php 程式出現 "No such file or directory" 錯誤

        apt-get install php5-cli

    如果執行 php 程式出現有關 ^M 的問題

        apt-get install dos2unix
        dos2unix message.php

    使用 cronjob 的時候, 如果安裝方式是使用 phpbrew, 那麼 php 來源要指定

        */1 * * * *
        CONSOLE_MESSAGE=/var/www/console-message-tool
        /usr/bin/env php $CONSOLE_MESSAGE/message.php check-system-info m="$(php $CONSOLE_MESSAGE/app/bin/get-system-info.php)"
